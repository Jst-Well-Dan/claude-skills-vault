## 💡 核心设计理念

### 从"技术视角"到"用户需求视角"

**原方案**（技术视角）：
```
工具介绍 → 技术对比 → 功能说明
```

**新方案**（用户需求视角）：
```
用户痛点 → 能做什么(Skill) → 怎么做到的(Claude Code) → 为什么更好(Agent特性) → 如何上手
```

### 语言策略

**必须做到**：
- ✅ 用生活化类比（助手、工具箱、流水线、App Store）
- ✅ 用具体场景（不说"提高效率"，说"从 2 小时减少到 5 分钟"）
- ✅ 用问答式标题（"为什么...?" "如何...?"）
- ✅ 强调结果而非过程（"自动生成报告"而非"调用 API 执行脚本"）

**必须避免**：
- ❌ 技术术语（API、JSON、YAML、frontmatter）
- ❌ 抽象描述（"强大的能力" → 改成"能做什么具体的事"）
- ❌ 程序员思维（"配置环境" → 改成"安装软件"）
